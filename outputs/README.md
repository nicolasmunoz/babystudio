# Outputs

Completed deliverables organized by type.

## Structure

```
/outputs
├── /content      # Written content
├── /strategy     # Strategy documents
├── /briefs       # Content briefs
├── /calendars    # Editorial calendars
├── /research     # Research and analysis
└── /assets       # Images, files
```

## Naming

Use descriptive names: `[type]-[description]-[date-if-relevant].md`

Example: `strategy-q1-marketing-plan.md`

## Linking

Always link outputs in the related GitHub Issue comment.
